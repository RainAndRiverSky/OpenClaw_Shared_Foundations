OpenClaw Shared Foundations

OpenClaw Shared Foundations provides the common governance, identity, memory, recovery, and stewardship layers used across OpenClaw systems.

Rather than duplicating foundational architecture across repositories, OpenClaw projects inherit shared principles, documentation standards, operational governance, and stewardship practices from this repository.

⸻

Purpose

This repository exists to provide:

* Governance foundations
* Identity architecture
* Memory architecture
* Recovery procedures
* MCP integration guidance
* Obsidian knowledge management structures
* Stewardship standards
* Cross-project operational consistency

These foundations allow multiple OpenClaw projects to evolve independently while remaining aligned to a common governance framework.

    Design Principles

OpenClaw Shared Foundations follows several principles:

Reusable

Foundations should be reusable across multiple repositories.

Implementation Neutral

Governance and architecture should not depend on a specific model, vendor, framework, or runtime.

Stewardship First

Human stewardship remains the ultimate authority for system evolution.

Documentation Driven

Governance should be observable, reviewable, and maintainable.

Recoverable

Systems should support continuity, restoration, and operational resilience.
